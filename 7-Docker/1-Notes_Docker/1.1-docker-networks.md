## üîë Docker Network Drivers (With Real Use Cases)

---

### 1Ô∏è‚É£ Bridge Network (Default)

- Default network driver in Docker
- Best for **single-host** container communication
- Containers can resolve each other using **container names (DNS)**

#### When to use:

- Simple applications
- 2-tier / 3-tier architecture
- Local development & testing

#### Real-world example:

3-tier architecture:

- Frontend
- Backend
- Database

#### Commands:

```bash
docker network create app_network

docker run --name db --network app_network -d mysql:5.7
docker run --name backend --network app_network -d backend:latest
docker run --name frontend --network app_network -d frontend:latest
```

‚úî Containers communicate using names like `db`, `backend`

---

### 2Ô∏è‚É£ Host Network (High Performance)

- Container **shares the host‚Äôs network stack**
- No NAT or port mapping
- Container listens directly on host ports

#### When to use:

- Performance-critical workloads
- Monitoring tools (Prometheus, Grafana)
- Network-intensive applications

#### Example:

```bash
docker run --network host prometheus
```

#### ‚ö†Ô∏è Risks:

- No network isolation
- Container can access host interfaces directly
- Security risk in production

> Use only when performance outweighs isolation.

---

### 3Ô∏è‚É£ Overlay Network (Multi-Host)

- Designed for **multi-host** container communication
- Used in:

  - Docker Swarm
  - Kubernetes (via CNI)

- Allows containers on different hosts to communicate as if on the same network

#### When to use:

- Cluster environments
- Distributed microservices
- Swarm services

#### Real-world example:

- Global service in Docker Swarm
- Communication between manager and worker nodes

‚úî Overlay abstracts networking across hosts

---

### 4Ô∏è‚É£ None Network (No Networking)

- Completely disables networking
- No IP address
- No network interface

#### When to use:

- Batch jobs
- Security-isolated workloads
- Containers that don‚Äôt require networking

#### Example:

```bash

```

‚úî Container runs in total isolation

## üõ†Ô∏è Basic Docker Network Commands

### Create a network

```bash
docker network create network-name
```

### List networks

```bash
docker network ls
```

### Connect container to network

```bash
docker network connect network-name container-name
```

### Disconnect container from network

```bash
docker network disconnect network-name container-name
```

### Inspect a network

```bash
docker network inspect network-name
```

### Remove a network

```bash
docker network rm network-name
```

### Remove unused networks

```bash
docker network prune
```
