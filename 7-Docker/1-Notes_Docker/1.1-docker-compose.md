## ‚úÖ **Basic Commands**

| Command                               | Description                                         |
| ------------------------------------- | --------------------------------------------------- |
| `docker compose up`                   | Builds (if needed), then starts all services        |
| `docker compose up -d`                | Runs in detached mode (background)                  |
| `docker compose down`                 | Stops and removes all containers, networks, volumes |
| `docker compose stop`                 | Stops running containers but keeps them             |
| `docker compose start`                | Starts previously stopped containers                |
| `docker compose restart`              | Restarts services                                   |
| `docker compose build`                | Builds or rebuilds services                         |
| `docker compose run <service>`        | Run a one-off command (like `exec`)                 |
| `docker compose exec <service> <cmd>` | Run a command in a running container                |
| `docker compose ps`                   | Shows running services and port mappings            |
| `docker compose logs`                 | Shows logs for all services                         |
| `docker compose logs -f`              | Follow logs live (like `tail -f`)                   |
| `docker compose config`               | Validates and prints merged config from YAML file   |

## üß™ **Advanced Commands**

| Command                                     | Description                                |
| ------------------------------------------- | ------------------------------------------ |
| `docker compose pull`                       | Pulls latest images                        |
| `docker compose push`                       | Pushes built images to a registry          |
| `docker compose rm`                         | Removes stopped service containers         |
| `docker compose top`                        | Shows processes in each container          |
| `docker compose version`                    | Show Compose version                       |
| `docker compose events`                     | Stream real-time events from containers    |
| `docker compose cp <service>:<path> <host>` | Copy files between container and host      |
| `docker compose create`                     | Creates containers but does not start them |

## üîÅ **Scale Services**

```bash
docker compose up --scale <service>=<count>
```

‚úÖ Example:

```bash
docker compose up --scale web=3
```

Runs **3 instances** of the `web` service.

---

## üßπ **Clean Up (System Level)**

```bash
docker compose down --volumes --remove-orphans
```

| Flag               | Description                                            |
| ------------------ | ------------------------------------------------------ |
| `--volumes`        | Deletes named/anonymous volumes                        |
| `--remove-orphans` | Removes containers not defined in `docker-compose.yml` |

---

## üì¶ **Docker Compose With Multiple Files**

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml up
```

Useful for overriding base config in **dev vs prod**.
